Summary A — High-level

Backend code: Echo-based Go services expose HTTP handlers (router.go → handlers → DTOs) that call service-layer business logic and repository functions; services use sqlc-generated repository code for db access.
Frontend: React + TypeScript (Vite) consumes the backend JSON DTOs, maps API responses to typed UI models, and drives user interactions; dev server runs natively while backend runs in Docker.

Backend code: The Go backend compiles to a single binary and uses Echo for routing and JSON/CORS handling. Handlers translate incoming JSON to DTOs and delegate to service functions; services implement inventory inference and call repository methods generated by sqlc to run migrations/queries. Docker + Air are used for dev/workflow.
Frontend: The web/ React TypeScript app expects stable typed DTOs from the backend (price → item mapping, transactions, product catalog). UI components and state rely on consistent API contracts; Vite provides fast HMR during local development while production is served as static assets.
